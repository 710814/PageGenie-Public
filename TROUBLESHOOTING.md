# 🔧 문제 해결 가이드

## 화면에 아무것도 보이지 않는 경우

### 해결 방법

1. **개발 서버 재시작**
   ```bash
   # 기존 서버 종료 (Ctrl+C)
   npm run dev
   ```

2. **브라우저 콘솔 확인**
   - F12 또는 Cmd+Option+I (Mac)로 개발자 도구 열기
   - "Console" 탭에서 오류 메시지 확인
   - 빨간색 오류가 있다면 스크린샷을 찍어 확인

3. **브라우저 새로고침**
   - 강력 새로고침: Ctrl+Shift+R (Windows) 또는 Cmd+Shift+R (Mac)

4. **캐시 삭제**
   - 브라우저 설정에서 캐시 삭제
   - 또는 시크릿 모드에서 테스트

### 최근 수정 사항

- ✅ 코드 스플리팅 문제 수정 (lazy import 제거)
- ✅ importmap 제거 (Vite와 충돌 방지)

## 일반적인 오류 및 해결 방법

### 1. "Cannot find module" 오류

**원인**: 의존성 설치 문제

**해결**:
```bash
rm -rf node_modules package-lock.json
npm install
```

### 2. "Failed to fetch" 오류

**원인**: API 키 또는 GAS URL 설정 문제

**해결**:
1. 설정에서 GAS URL 확인
2. Apps Script 프로젝트 배포 확인
3. API 키가 스크립트 속성에 설정되었는지 확인

### 3. "React is not defined" 오류

**원인**: React import 문제

**해결**: 이미 수정됨. 개발 서버 재시작

### 4. 화면은 보이지만 스타일이 깨진 경우

**원인**: Tailwind CSS 로드 문제

**해결**:
- `index.html`에 Tailwind CDN이 포함되어 있는지 확인
- 브라우저 새로고침

## 개발 서버 실행 확인

정상적으로 실행되면 다음과 같은 메시지가 표시됩니다:

```
  VITE v6.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: http://192.168.x.x:3000/
```

이 URL로 접속하면 애플리케이션이 표시되어야 합니다.

## 추가 디버깅

### 브라우저 콘솔에서 확인할 사항

1. **네트워크 탭**: 리소스 로딩 실패 확인
2. **콘솔 탭**: JavaScript 오류 확인
3. **요소 검사**: `#root` 요소가 있는지 확인

### 터미널에서 확인할 사항

1. **빌드 오류**: 컴파일 오류 메시지 확인
2. **포트 충돌**: 다른 프로세스가 3000 포트를 사용 중인지 확인

---

## 여전히 문제가 있는 경우

1. 브라우저 콘솔의 오류 메시지 전체를 복사
2. 터미널의 오류 메시지 전체를 복사
3. 문제를 보고할 때 이 정보를 포함
