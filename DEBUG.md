# 디버깅 가이드

## 화면이 보이지 않는 경우 체크리스트

### 1. 브라우저 콘솔 확인 (가장 중요!)

1. **F12** 또는 **Cmd+Option+I** (Mac)로 개발자 도구 열기
2. **Console** 탭 확인
3. 빨간색 오류 메시지가 있는지 확인
4. 오류 메시지를 복사해서 알려주세요

### 2. 네트워크 탭 확인

1. 개발자 도구의 **Network** 탭 열기
2. 페이지 새로고침 (F5)
3. 실패한 요청(빨간색)이 있는지 확인
4. 특히 `.js`, `.css` 파일이 로드되지 않았는지 확인

### 3. Elements 탭 확인

1. 개발자 도구의 **Elements** (또는 **Inspector**) 탭 열기
2. `<div id="root">` 요소가 있는지 확인
3. `#root` 안에 내용이 있는지 확인

### 4. 개발 서버 확인

터미널에서 다음을 확인:
- `npm run dev` 명령어가 실행 중인지
- 오류 메시지가 있는지
- "VITE ready" 메시지가 표시되는지

### 5. 브라우저 주소 확인

- `http://localhost:3000` 접속했는지 확인
- 다른 포트를 사용 중인지 확인 (터미널에 표시된 포트 확인)

## 일반적인 문제와 해결

### 문제 1: "Cannot find module" 오류

**해결**:
```bash
rm -rf node_modules
npm install
```

### 문제 2: "React is not defined" 오류

**해결**: 이미 수정됨. 개발 서버 재시작

### 문제 3: CSS가 적용되지 않음

**해결**: 
1. `index.css` 파일이 있는지 확인
2. Tailwind가 제대로 설치되었는지 확인
3. 브라우저 강력 새로고침 (Ctrl+Shift+R)

### 문제 4: 완전히 빈 화면

**가능한 원인**:
- JavaScript 오류로 인한 렌더링 실패
- CSS가 로드되지 않아서 흰 화면
- React 컴포넌트가 렌더링되지 않음

**해결**: 브라우저 콘솔의 오류 메시지 확인
